## Decision Motivations — The Role Matrix

Decision motivations are organized by the role someone plays in a team. The matrix categorizes people by three dimensions: **Postulation** (setting goals/strategy), **Optimization** (specifying methods), and **Realization** (execution).

| Role | Postulation (Strategy) | Optimization (Methods) | Realization (Execution) |
|------|----------------------|----------------------|----------------------|
| **Investor** | ✓ | ✗ | ✗ |
| **Entrepreneur** | ✓ | ✓ | ✗ |
| **Cooperator** | ✓ | ✗ | ✓ |
| **Solo Entrepreneur / Lone Wolf** | ✓ | ✓ | ✓ |
| **Advisor** | ✗ | ✓ | ✗ |
| **Manager** | ✗ | ✓ | ✓ |
| **Worker** | ✗ | ✗ | ✓ |
| **Supervisory Board / State Treasury Rep** | ✗ | ✗ | ✗ |

### Role Descriptions

- **Investor** — Postulates (determines strategy, gives direction) but does not specify methods or execute.
- **Entrepreneur** — Sets goals AND works out methods, optimizes processes, but does not execute.
- **Cooperator** — Sets goals and executes but does not optimize. Common in old systems where many postulated, many executed, but few optimized — leading to collapse.
- **Solo Entrepreneur / Lone Wolf** — Does everything: sets goals, specifies methods, and executes.
- **Advisor** — Optimizes methods for the investor or entrepreneur but does not postulate or execute. Common consulting role.
- **Manager** — Optimizes and executes but does not define strategy.
- **Worker** — Simply executes. An employee.
- **Supervisory Board / State Treasury Reps** — None of the three. Their role lies in **cognitive processes** — processing the transcendent limit, interpreting information from the broader system to enable higher-level organization.

### Key Insight: Poles Confuse Goals with Methods

A common steering error: people with a predisposition for specifying methods call their method a "strategy." This methodological confusion was historically conditioned — society was trained to focus only on executing external organizers' goals rather than setting their own. Restoring the ability to set goals at the individual, state, and national level is crucial.

## Cognitive Motivations — The Three Types

Cognitive motivations are what happens **before** any decision. They involve understanding what IS, not what SHOULD BE. Three sub-processes:

### 1. Exploration
The **Traveler** or **Explorer** — someone who goes to see what's there. They look around corners, discover what exists. In business: researching what competitors are doing. This is **philosophical** — they see things at the level of broad patterns.

### 2. Classification
Once information is gathered, someone with classification talent organizes it. They create categories, describe elements, separate things into clean groups. "Company A makes nails, Company B hammers nails, Company C ships nails."

### 3. Explication
Someone with explication talent connects everything with relationships. They determine the structure: which elements relate to which, how they form a functional whole. Only after all three work together can you understand the complete picture.

**Important:** These three involve **no decisions**. The Explorer explored — didn't decide. The Classifier classified — didn't decide. The Explicator connected — didn't set any goal. These people are essential but often overlooked, especially by execution-focused individuals who see cognitive work as just "that smiley face."

## The Logic Gate of Mazur-Kosecki

The logic gate shows how cognitive and decision processes are inseparably linked:

| Component | Process |
|-----------|---------|
| **Input state = System X** | Must know what X is |
| **Elements X1, X2** | Classification |
| **Relations between elements** | Information in set X |
| *(These three = Cognitive Process)* | |
| **Postulation = Set Y** | Defining the goal (what SHOULD be) |
| **Method** | Transformation from X to Y |
| **Substrate** | Starting point — what's available |
| *(These three = Decision Process)* | |

The logic gate: input (resources) → operation (code) → output (goal). It gives the most complete picture of the control process.

## The General Theory of Motivation — All 12 Types

The control process has **6 sub-processes**:
1. **Exploration** — discovering what exists
2. **Classification** — organizing elements
3. **Explication** — understanding connections
4. **Postulation** — defining goals
5. **Optimization** — choosing methods
6. **Realization** — executing with resources

Each of these 6 can be triggered in **two ways**: by **external influence** (environment presents an opportunity) or by **internal reflection** (self-initiated through feedback loops).

### Decision Motivations (6)

| Motivation | Control Stage | Trigger | Description |
|------------|--------------|---------|-------------|
| **Ideological** | Postulation | Internal | Goals that come from within |
| **Adaptive** | Postulation | External | Goals triggered by the environment |
| **Ethical** | Optimization | Internal | Methods chosen by personal values |
| **Legal / Coercive** | Optimization | External | Methods forced by environment |
| **Vital** | Realization | Internal | Using internal resources |
| **Economic** | Realization | External | Using external resources |

### Cognitive Motivations (6)

Each cognitive sub-process splits by direction:

| Motivation | Control Stage | Direction | Description |
|------------|--------------|-----------|-------------|
| **Meta-exploration** | Exploration | Upward | Seeking connection to larger wholes |
| **Sub-exploration** | Exploration | Downward | Looking inward into subsystems |
| **Meta-classification** | Classification | Upward | Classifying in relation to larger systems |
| **Sub-classification** | Classification | Downward | Classifying in relation to smaller systems |
| **Meta-explication** | Explication | Upward | Seeking higher-order connections |
| **Sub-explication** | Explication | Downward | Seeking connections downward |

## Why All 12 Must Be Present

Remove any one of the 12 and the **autonomous process system breaks**:

- No internal ideas (only reacting to environment) → someone else is your organizer
- No external adaptation (only own ideas) → first disturbance eliminates you
- No own methods → you become a system organized by someone else
- No cognitive sub-processes → the image of input system X is incomplete, making autonomous control impossible

Creating an **autonomous** (not just self-steering) system requires all 12 motivation types to be filled.

## Semantic Precision — Avoiding Info-Viruses

A critical methodological warning: words must be defined unambiguously. When the same word has multiple designata (referents), people think they're discussing the same thing when they're actually talking about different concepts. This creates **info-viruses, normo-viruses, and code-viruses** — deliberate disinformation methods used to confuse the distinction between goal and method, making it impossible for people to objectively steer according to the laws of nature.
